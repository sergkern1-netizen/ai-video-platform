# Улучшение качества рендеринга видео — дизайн

Дата: 2026-06-18

## Проблема

Текущая реализация (`backend/pipeline/video_renderer.py`, `asset_fetcher.py`, `script_generator.py`) рендерит видео из **одного зацикленного клипа** на весь ролик, без переходов, с искажением пропорций (наивный `resize`), субтитры иногда выглядят с ошибками (приближенные тайминги по словам, фолбэк на `ImageFont.load_default()` если `arial.ttf` не найден). Видео выглядит непрофессионально и не подходит для привлечения аудитории на YouTube/TikTok.

## Цель

Сделать генерируемые видео визуально и звуково качественными, органичными для зрителя, с форматом, заточенным под платформу (YouTube long-form 16:9 vs TikTok/Shorts/Reels 9:16), независимо от темы ролика.

## Архитектура / поток данных

```
GPT (формат-aware промпт) → Script{scenes: [Scene{text, keywords, duration_sec}], mood, ...}
                                           │
                    ┌──────────────────────┴───────────────────────┐
                    ▼                                               ▼
     TTS(full body) → audio.mp3 → Whisper(audio) → word_timings    fetch_assets(scenes) → 1 клип на сцену
                                                                     │ (orientation-aware Pexels query)
                                                                     ▼
                                          render_video: crop-to-fill + Ken Burns + crossfade
                                                                     │
                                          + word-by-word караоке-субтитры (синхронные по Whisper)
                                                                     │
                                          + фоновая музыка (подбор по mood, duck под голос)
                                                                     ▼
                                                              output/<id>.mp4
```

## Компоненты

### 1. `script_generator.py`

GPT возвращает структуру со сценами вместо плоского `body`:

```json
{
  "title": "...",
  "scenes": [{"text": "...", "keywords": ["...", "..."], "duration_sec": 4}],
  "mood": "upbeat|calm|dramatic|corporate",
  "duration_sec": 50
}
```

- Промпт `short`: явный хук в первой сцене (1-2 сек завязки), сцены 3-5 сек каждая, динамичный темп.
- Промпт `long`: плавное вступление, сцены 8-15 сек, более спокойный темп.
- `mood` — для выбора фоновой музыки (одно из фиксированного набора значений, соответствующих файлам в `backend/assets/music/`).
- `Script.body` — склейка текста всех сцен по порядку (единое связное аудио из TTS, без швов в голосе).
- `Script` датакласс: `title: str`, `scenes: list[Scene]`, `body: str`, `mood: str`, `duration_sec: int`. `Scene`: `text: str`, `keywords: list[str]`, `duration_sec: int`.

### 2. `voice_synthesizer.py`

После генерации TTS-аудио (`client.audio.speech.create`), дополнительно вызывается:

```python
client.audio.transcriptions.create(
    model="whisper-1",
    file=open(audio_path, "rb"),
    response_format="verbose_json",
    timestamp_granularities=["word"],
)
```

Реальные тайминги слов (`{word, start, end}`) заменяют текущую оценку через равномерное деление по группам из 8 слов. При ошибке Whisper — fallback на текущую оценочную логику (`_estimate_timings`), пайплайн не должен падать из-за этого.

`VoiceOutput.word_timings` теперь хранит по одной записи на **слово** (не на чанк из 8 слов) — нужно для word-by-word караоке.

### 3. `asset_fetcher.py`

`fetch_assets(scenes: list[Scene], format: Literal["short", "long"]) -> Assets` — для каждой сцены отдельный запрос в Pexels:
- `orientation="portrait"` для `short`, `orientation="landscape"` для `long`.
- Запрос по `scene.keywords` (первые 2 слова).
- Если для сцены ничего не найдено — fallback на клип предыдущей успешной сцены; если это первая сцена и фолбэка нет — используется нейтральный цветной фон (текущее поведение `ColorClip`).

`Assets.video_clips` — список длиной = количеству сцен, каждый элемент `{path, duration_sec}` привязан к конкретной сцене по индексу.

### 4. `video_renderer.py`

Для каждой сцены:
1. Взять клип сцены, **crop-to-fill** под целевой размер (обрезка по центру до точного аспекта без искажения пропорций) — заменяет текущий наивный `resize`.
2. Подрезать/зациклить под `scene.duration_sec`.
3. Применить лёгкий Ken Burns эффект: медленный zoom 1.0 → 1.08 на протяжении длительности сцены (через `resize` с функцией от времени или `fl` трансформацию кадра).
4. Между соседними сценами — crossfade 0.4с (`crossfadein`/`crossfadeout`, сцены должны перекрываться по таймлайну на длительность перехода).

Субтитры — word-by-word караоке: на экране короткая группа слов (текущее окно), активное слово выделено (цвет/масштаб), синхронизировано с `word_timings` от Whisper. Шрифт — бандленный `.ttf` (например `backend/assets/fonts/Montserrat-Bold.ttf`), без фолбэка на системные шрифты.

Аудио: голос + фоновый трек из `backend/assets/music/<mood>.mp3`, громкость трека ~15-20% от голоса, fade-in/fade-out на старте/конце ролика. Если длительность музыки короче видео — зациклить.

Качество экспорта: явный `bitrate` вместо дефолтного — `"8000k"` для short (1080×1920), `"6000k"` для long (1920×1080), `fps=30`, `codec="libx264"`, `audio_codec="aac"`.

### 5. Музыкальная библиотека

`backend/assets/music/` — 5-8 коротких (1-3 мин, зацикливаемых) CC0/royalty-free треков, по одному на каждое значение `mood` (`upbeat.mp3`, `calm.mp3`, `dramatic.mp3`, `corporate.mp3` и т.п.).

## Обработка ошибок

- Pexels не вернул клип для сцены → fallback на клип предыдущей сцены → если совсем нет → нейтральный `ColorClip`. Не прерывает пайплайн.
- Whisper transcription упал (сетевая ошибка/таймаут) → fallback на текущую оценку таймингов по длине слов. Логируется, но не прерывает пайплайн.
- Шрифт не найден в репозитории → это ошибка конфигурации, пайплайн должен упасть явно (не должно происходить в норме, т.к. шрифт версионируется в git).
- Музыкальный трек для `mood` не найден (опечатка/новое значение от GPT) → fallback на трек `corporate.mp3` (нейтральный дефолт).

## Тестирование

Юнит-тесты (без реальных API-вызовов, моки/чистые функции):
- Парсинг JSON-ответа GPT со сценами в `Script`/`Scene` (валидные и невалидные данные).
- Расчёт параметров Ken Burns (zoom-фактор от времени) и crossfade-окон по длительностям сцен — чистые функции, тестируемые без MoviePy рендера.
- Выбор фонового трека по `mood`, включая fallback на `corporate.mp3`.
- Fallback-логика `fetch_assets` при пустом ответе Pexels для одной/нескольких сцен.

Полный E2E (реальные GPT/TTS/Whisper/Pexels/MoviePy вызовы) — вручную, как в предыдущей сессии (Task 10), не автоматизируется юнит-тестами.
